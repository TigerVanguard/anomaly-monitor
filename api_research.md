# API 调研结果

## Polymarket API
- **费用**: 
  - 大多数市场**不收取交易费用**。
  - **15分钟加密货币市场**收取 Taker 费用，用于资助 Maker Rebates Program。
  - **API 访问本身通常是免费的**，包括核心的市场和价格端点。
- **速率限制**:
  - 免费层级通常有每分钟 100 次请求的限制。
  - 速率限制会随着交易量的增加而扩展（Builder Tiers）。
  - 所有的速率限制都通过 Cloudflare 的节流系统执行。
- **功能**:
  - 提供 REST API 用于获取市场信息（只读）。
  - 提供 CLOB (Central Limit Order Book) API 用于交易和订单管理。
  - 提供 Real Time Data Socket (RTDS) 用于实时数据订阅。

## Twitter API (X API)
- **费用**:
  - **Free Tier**: 仅用于写入用例和测试，功能非常有限。
  - **Basic Tier**: $200/月，适用于爱好者或原型开发，读取限制较严。
  - **Pro Tier**: 约 $5,000/月（根据 Reddit 讨论），也有第三方服务提供更便宜的替代方案（如 $24.99/月 100k 推文）。
  - **Enterprise Tier**: 起价 $42,000 - $50,000/月，提供大规模数据访问。
- **用途**:
  - **舆情监控**: 监控特定关键词（如 "Polymarket", "Trump", "Bitcoin"）的情绪变化。
  - **异常检测**: 检测突发新闻或社交媒体上的异常讨论热度，这往往是市场波动的前兆。
  - **内幕信息辅助**: 结合链上数据，验证某些大额投注是否与社交媒体上的内幕消息泄露有关。
  - **替代方案**: 由于官方 API 价格昂贵，许多开发者转向使用第三方数据提供商或爬虫服务（如 Apify, Bright Data 等）。

## 结论
- **Polymarket API** 是构建此系统的核心且成本低廉（主要是免费的，除非进行高频交易）。
- **Twitter API** 是关键的辅助数据源，但官方 API 成本极高。对于个人或小型项目，建议使用第三方数据服务或仅使用 Basic Tier 进行有限的关键词监控。

# Twitter 数据获取替代方案调研

## 1. Apify - Twitter (X.com) Tweets & Profiles Scraper
- **简介**: 强大的爬虫工具，无需 API 密钥，支持搜索、用户资料、推文及回复抓取。
- **定价**:
  - **月费**: $30.00/月（平台订阅费）+ 使用量费用。
  - **使用量费用**: 约 $0.25 - $0.40 / 1,000 条推文。
  - **免费试用**: 提供 $5 的免费额度，足以进行初步测试。
- **优点**:
  - **无需 Twitter 认证**: 规避了官方 API 的复杂认证和高昂费用。
  - **功能全面**: 支持关键词搜索、历史数据、媒体下载、回复抓取。
  - **多种导出格式**: JSON, CSV, Excel 等，方便集成。
  - **云端运行**: 无需本地维护爬虫环境和代理池。
- **缺点**: 需要支付 Apify 平台月费，对于极低频使用者可能不够灵活。

## 2. RapidAPI - Twitter/X API by Datarise
- **简介**: 通过 RapidAPI 市场提供的第三方 API，封装了 Twitter 数据抓取功能。
- **定价**:
  - **Basic**: $0.00/月（通常包含极少量请求用于测试）。
  - **Pro**: $9.99/月。
  - **Ultra**: $49.99/月。
  - **Mega**: $99.99/月。
- **优点**:
  - **按需付费**: 门槛低，适合个人开发者和小型项目。
  - **标准 API 接口**: 易于集成，类似官方 API 的使用体验。
- **缺点**: 依赖第三方服务的稳定性，可能受 Twitter 反爬策略影响。

## 3. 开源爬虫工具 (自建方案)
- **工具**: Nitter (自建实例), snscrape (已停止维护), twint (已停止维护), open-source scrapers on GitHub.
- **现状**: 由于 Twitter 反爬策略升级，大多数开源工具（如 snscrape, twint）目前已失效或极不稳定。
- **建议**: 不推荐作为生产环境的首选方案，维护成本极高，且容易导致 IP 被封禁。

## 推荐方案
对于本系统，**推荐使用 Apify**。
- **理由**: 稳定性高，维护成本低，按量付费模式适合监控系统的间歇性抓取需求。虽然有 $30 月费，但相比官方 API 的 $42,000+ 仍然极具性价比，且比自建爬虫更可靠。
- **集成方式**: 使用 Apify Client (Python/Node.js) 调用 Actor，通过 Webhook 将数据推送到监控系统。
